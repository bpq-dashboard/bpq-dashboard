WP Manager — Winlink White Pages Cleaner
python3 scripts/wp_manager.py --scan
Scan WP.cfg and write wp_review.txt — flags invalid callsigns, BBS names, stale entries (>730 days), ghost records, and hierarchy anomalies. AUTO-REMOVE entries pre-filled; others marked ? for review.

nano wp_review.txt
Edit review file — change ? to REMOVE or KEEP for each flagged entry. AUTO entries are pre-filled REMOVE; leave or override as needed.
edit
python3 scripts/wp_manager.py --apply
Apply decisions from wp_review.txt — surgically removes REMOVE entries (no renumbering), creates timestamped backup of WP.cfg, then automatically restarts BPQ via systemctl restart bpq. KEEP decisions added to whitelist.
apply + restart
python3 scripts/wp_manager.py --diff
Compare current WP.cfg against saved baseline — detects entries re-injected by Winlink CMS on the last sync. Run after each Winlink session to check for re-injection of previously removed callsigns.
diff
python3 scripts/wp_manager.py --blacklist-add CALLSIGN1 CALLSIGN2
Add persistent offenders to blacklist — always flagged on future scans even if CMS repopulates. Use for entries that keep re-appearing after removal.
blacklist
python3 scripts/wp_manager.py --auto-clean --dry-run
Preview automated clean — shows what would be removed (invalid callsigns, BBS names, blacklisted) without making any changes. Run this before scheduling as a cron job.
preview
python3 scripts/wp_manager.py --auto-clean
Automated clean — removes AUTO-flagged (invalid format, BBS names, empty) and BLACKLISTED entries immediately. STALE and REVIEW entries are never touched automatically. Creates backup, restarts BPQ, updates baseline.
auto remove
